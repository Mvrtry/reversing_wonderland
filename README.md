## שלב 2

נכנסנו לפונקציה שאחראית על השלב השני (בכתובת `00401330`).
כשאנחנו מסתכלים על הגרף, הבלוקים הראשונים מראים התעסקות די סטנדרטית של קבלת קלט.

התוכנית מקצה מקום בזיכרון – `buffer` ואז קוראת ל-`memset` בכדי לאפס אותו, משתמשת ב-`fgets` בכדי לקלוט את המחרוזת שלנו, ומיד קוראת ל-`strlen` בכדי לשמור את אורך הקלט במשתנה מקומי. יש לציין שהמערכת קוראת את הנתונים בגושים של 4 בתים (DWORD).

![קבלת קלט ואורך](נתיב/לתמונה_הקצאת_זיכרון.png)

במקום להתעכב על הפיצולים חיפשתי מה יכול להתחיל לולאה או התעסקות חריגה – בד"כ במקרים כאלה אני מעביר ממצב גרף למצב טקסט וככה אפשר לראות בבירור לולאות ואת סדר העבודה של המעבד:

![מצב טקסט לולאה](נתיב/לתמונה_לולאה_טקסט.png)

בהסתכלות מהירה אפשר לשים לב לשורה חריגה: ניתן לראות בבירור פעולת `XOR` עם מספר הקסדצימלי.
התוכנית עוברת על הקלט שלנו ב''דילוגים'' של 4 בתים (ע''פ `DWORD`).
אם נתרגם את הרצף המדובר לאסקי נקבל את המילה `ABRA` שהיא למעשה מילת המפתח למציאת הסיסמה.

אם נסתכל על בלוק היציאה מהלולאה נראה שאנחנו טוענים למעבד מחרוזת קבועה.

![מחרוזת קבועה](נתיב/לתמונה_יציאה_מלולאה.png)

משמע התוכנית שלנו בודקת את האינפוט אחרי שעבר `XOR` עם `ABRA` והאם שווה למחרוזת היעד (`INTO THE RABBIT HOLE`).
בגלל ש-`XOR` היא פעולה סימטרית נהפוך את הסדר ונחלץ בעזרת `XOR` של `INTO THE RABBIT HOLE` עם `ABRA` את הסיסמה.

ניעזר בסקריפט פייתון קצר לעזרה:

```python
target_string = "into the rabbit hole"
key = "ABRA"
password = ""

for i in range(len(target_string)):
    char_code = ord(target_string[i]) ^ ord(key[i % len(key)])
    password += chr(char_code)

print(password)
